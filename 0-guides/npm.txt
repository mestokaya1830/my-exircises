📦 NPM Package Management Master Guide
🔍 1. Inspecting Packages
Check what is installed and find out what needs an update.

List Project Packages: npm list

List Global Packages: npm list -g

Check for Updates: npm outdated (Shows current vs. latest versions)

Check Global Updates: npm outdated -g --depth=0 (Focuses only on top-level tools)

🆙 2. Updating Packages
Keep your tools and dependencies on the latest stable versions.

Update Project: npm update (Updates based on package.json ranges)

Update Global Tools: npm update -g

Update NPM itself: npm install npm@latest -g 🚀

🛡️ 3. Maintenance & Security (Essential Additions)
These commands help you fix broken projects and keep them secure.

Fix Security Vulnerabilities: npm audit fix

Checks your tree for known risks and updates the risky packages automatically.

Clean NPM Cache: npm cache clean --force

Use this if an installation keeps failing for no apparent reason.

Remove a Package: npm uninstall <package-name>

Check Why a Package is Installed: npm explain <package-name>

Very useful for finding out which "hidden" dependency required a specific library.

💡 Pro Tip: The npx Power Move
Instead of installing every tool globally (like pkg or create-vite), you can run them once without taking up permanent disk space:

Run without installing: npx <package-name> (e.g., npx cowsay hello)
