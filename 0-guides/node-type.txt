🚀 Professional Express & TypeScript Setup Guide
This setup uses tsx for a fast development experience and strict TypeScript rules for high-quality code.

1️⃣ Environment & Dependency Setup
Initialize your project and install the necessary tools.

Global Installation:

📦 npm i -g typescript

✅ tsc -v

Project Initialization:

📁 npm init -y

Production Dependencies:

💾 npm install express cors dotenv

Development Dependencies:

🛠️ npm install -D typescript @types/node @types/express @types/cors tsx

2️⃣ TypeScript Configuration
Create and configure the behavior of the compiler.

⚙️ tsc --init

📝 Open tsconfig.json and replace the content with these strict settings:

JSON
{
  "compilerOptions": {
    "rootDir": "./src",            // Source files location
    "outDir": "./dist",           // Compiled JS output folder
    "module": "nodenext",         // Hybrid ESM/CJS system
    "target": "esnext",           // Modern JS for latest Node.js
    "types": ["node"],            // Load Node.js types
    "lib": ["esnext"],            // Standard JS libraries
    "strict": true,               // Enable all strict type checking
    "noUncheckedIndexedAccess": true,   // Safety for array access
    "exactOptionalPropertyTypes": true, // Prevent undefined on optional fields
    "verbatimModuleSyntax": true,       // Enforce 'import type'
    "isolatedModules": true,            // Independent file compilation
    "moduleDetection": "force",         // Treat all files as modules
    "skipLibCheck": true                // Ignore errors in node_modules
  }
}
3️⃣ Scripts & Folder Structure
Prepare your project for development and production.

📂 Create Folder: mkdir src

📄 Update package.json: Add these scripts to manage your app:

JSON
"scripts": {
    "dev": "tsx watch src/index.ts",
    "start": "node dist/index.js",
    "build": "tsc"
}
4️⃣ The Application Code
Create your entry point: touch src/index.ts and paste the logic below.

TypeScript
import express, { Request, Response, NextFunction } from 'express';
import type { ErrorRequestHandler } from "express";
import cors from 'cors';
import dotenv from 'dotenv';

dotenv.config();

const app = express();
app.use(cors());
app.use(express.json());

// Routes
app.get('/', (req: Request, res: Response) => {
  const users = [
    { id: 0, full_name: 'Mustafa Kaya', age: 45, active: true },
    { id: 1, full_name: 'Ekrem Kaya', age: 23, active: true }
  ];
  res.json(users);
});

// 404 Handler
app.use((req: Request, res: Response) => {
  res.status(404).json({ error: "404 Page Not Found" });
});

// Global Error Handler
const errorHandler: ErrorRequestHandler = (err, req, res, next) => {
  console.error(err);
  res.status(500).json({
    success: false,
    message: err.message || "Internal Server Error"
  });
};
app.use(errorHandler);

// Server Start
const start = async () => {
  const PORT = Number(process.env.PORT) || 3000;
  app.listen(PORT, () => {
    console.log(`🚀 Server running at http://localhost:${PORT}`);
  });
};

start();
5️⃣ Execution
Now you are ready to launch!

🔥 Start Development: npm run dev
(Uses tsx to auto-restart on changes)

🏗️ Build for Production: npm run build
(Generates the dist folder)

🚀 Run Production: npm start
